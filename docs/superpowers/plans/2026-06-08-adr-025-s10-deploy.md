## Transport (v1.2 update per ADR-031)

**SSH inbound is structurally blocked by Azure Policy (service principal `7355f99c-...`).** All deploy commands below run via `az vm run-command invoke`:

```powershell
# Helper PowerShell function used throughout v1.2:
function Invoke-VmShell { param([string]$Script)
  $tmp = New-TemporaryFile
  $Script | Out-File -Encoding ascii $tmp
  az vm run-command invoke -g RG-NANOCLAW-AZURE -n nanoclaw-az `
    --command-id RunShellScript --scripts "@$tmp" --query "value[0].message" -o tsv
  Remove-Item $tmp
}
```

Throughout this plan, `ssh nanoclaw-az "<CMD>"` should be read as `Invoke-VmShell '<CMD>'`. The run-command extension serializes per-VM (~30s per call incl. warmup), so allow ~60 min total wall clock for D0–D6 instead of the original 30 min. Only ONE run-command can be in flight per VM — if another session holds the lease, you'll see `Conflict: Run command extension execution is in progress`. Wait + retry.
